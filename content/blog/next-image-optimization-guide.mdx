# Next.js Image Optimization Guide

From responsive sizes to `priority` and `placeholder`, these defaults will keep images crisp and fast.

## The Essentials

- Always set an appropriate `sizes` attribute to avoid over-fetching
- Use `fill` for container-based layouts; `width`/`height` for intrinsic
- Only set `priority` on above-the-fold hero media
- Prefer `placeholder="blur"` with a `blurDataURL` for LCP images

```tsx
<Image
  src={cover}
  alt={title}
  width={1200}
  height={800}
  sizes="(max-width:768px) 100vw, 50vw"
  priority
  placeholder="blur"
  blurDataURL={lqip}
/>
```

## Remote Patterns

Configure `next.config.mjs` to allow remote hosts:

```js
images: {
  remotePatterns: [
    { protocol: 'https', hostname: 'images.unsplash.com' },
    { protocol: 'https', hostname: 'images.pexels.com' },
  ],
},
```

## Common Pitfalls

- Oversized `sizes` attribute causing 2× bandwidth
- Missing `alt` or misleading descriptions
- `priority` used on all images (hurts CPU/network)

## Debugging

Use Chrome DevTools > Network to verify requested dimensions and format. Aim for modern formats (AVIF/WEBP) and <100KB for list thumbnails.
